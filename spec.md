# 即時惡意內容防護擴充功能
**系統架構與實作規格書 (System Architecture Specification)**

## 1. 系統概述 (System Overview)
本系統旨在提供網頁瀏覽環境中的「零時差」惡意行為防護。結合瀏覽器擴充功能（Client-Side）與邊緣運算（Edge AI）技術，系統能即時獲取 DOM 結構進行特徵分析，並利用輕量級的 WebLLM（0.5B - 1B 參數）進行本地端推論，最終產出目標網頁或區塊的**信用評分 (Trust/Credit Score)**。在面臨高度混淆或深度推論需求時，系統具備雙軌切換機制，可將分析任務路由至雲端或企業端的高性能模型。

---

## 2. 核心運作流程 (Core Workflow)

1. **DOM 解析與特徵萃取 (Client-Side)**
   擴充功能即時監聽網頁上的高敏感交互元件（如 Input 框、Button 等）。當觸發敏感交互時，啟動「結構化特徵萃取器」，將龐大的 HTML DOM 壓縮為輕量級的結構化特徵（例如移除無效樣式、提取網格座標與行為標籤），減輕後續模型的運算負擔。

2. **本地特徵快取比對 (Local Cache Lookup)**
   萃取出的結構特徵會優先送往「本地特徵快取資料庫 (3NF)」。若特徵匹配已知的惡意或安全指紋，則直接命中快取，大幅降低延遲並減少 AI 推論的資源消耗；若快取未命中，則進入 AI 推論階段。

3. **WebLLM 輕量化邊緣推論 (Edge AI Inference)**
   採用 0.5B 至 1B 參數級別的 **WebLLM** 作為核心推理引擎。WebLLM 直接於用戶本地端運行（利用 WebGPU 加速），分析潛在問題區塊或整體網頁行為，保障隱私數據零外流。

4. **信用評分與阻斷機制 (Scoring & Blocking)**
   WebLLM 分析完成後，會給出該區塊或網站的「信用評分」。若評分低於安全閾值（確信為惡意行為），系統將觸發「惡意行為攔截與預警介面」進行即時阻斷。若判定結果模糊或涉及高度混淆代碼，則透過「雙軌切換路由器」進行動態分流。

## 3. 系統架構藍圖 (Architecture Components)

根據系統架構設計圖，整體防護分為三道防線：

| 防線層級 | 模組名稱 | 功能說明 |
| :--- | :--- | :--- |
| **第一道防線**<br>(Client-Side) | 觸發式運行模組<br>結構化特徵萃取器 | 即時監控 DOM 交互，並將複雜的網頁結構降維，提取屬性層級與行為模式特徵。此層次包含攔截與預警介面。 |
| **核心運算**<br>(Edge AI) | 本地特徵快取資料庫<br>輕量化 AI 推論引擎 (WebLLM)<br>雙軌切換路由器 | 執行本地快取查詢。未命中時由 0.5B-1B 的 WebLLM 產出信用評分。路由器負責依據推論深度需求，動態切換至雲端模型。 |
| **深度防禦**<br>(Cloud/Enterprise) | 企業私有雲 LLM (Ollama)<br>官方託管高性能 AI 引擎 | 處理高難度分析任務。企業版資料留在內部網段；專業版訂閱則透過 Google Cloud Secret Manager 進行 API 金鑰安全管控。 |

> **💡 關鍵技術指標：0.5B - 1B WebLLM 部署優勢**
> 選擇 0.5B 至 1B 參數的模型大小，能在普通消費級硬體（如筆電整合顯卡）上透過 WebGPU 實現極低延遲的推論。這確保了在生成「信用評分」時，不會造成網頁載入或使用者交互的卡頓，同時免除了雲端 API 往返的時間成本與隱私外洩風險。

---

## 4. 信用評分與判定邏輯 (Scoring Logic)

WebLLM 的輸出將被格式化為標準的 JSON 評估報告，主要包含：

* **信用評分 (Credit Score)**：0 至 100 分。分數越低代表威脅程度越高。
* **威脅類型 (Threat Category)**：如 XSS 注入、釣魚表單、挖礦腳本或惡意導向。
* **置信度 (Confidence)**：推論的把握程度。若置信度低於設定標準，觸發雙軌切換路由器，將結構化特徵轉送至雲端或 Ollama 進行二次深度驗證。
